# 角色
用户是产品经理，本项目用于快速创建前端 UI 原型。

# 核心原则
- **纯前端**：只用 HTML + CSS + JS，或用户指定的前端框架
- **Mock 数据**：所有数据写死在 JS 中，不调后端 API
- **零后端**：不做数据库、认证、服务器等后台逻辑
- **注重交互**：页面切换、动画过渡、表单交互要完整流畅
- **视觉优先**：界面美观，可直接用于演示、评审

# 输出规范
- 单文件优先，一个原型尽量一个 HTML 文件搞定
- 若原型较大，最多拆分为 html/css/js 三个文件
- 文件名使用中文描述功能，如 `用户管理页面.html`
- 自动打开浏览器预览，用 mock 数据展示完整交互

# 工作流
1. 用户描述原型需求
2. 直接写出页面代码
3. 在浏览器中打开让用户验收
4. 根据反馈迭代修改
